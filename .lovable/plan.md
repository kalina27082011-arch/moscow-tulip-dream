В админке уже есть раздел «Букеты» (`/admin/products`) с полным набором действий: создание нового букета, редактирование карточки (название, slug, цена, описание, состав, цвет, изображение, сортировка, видимость), скрытие и удаление. Похоже, раздел просто не на виду — в верхней навигации админки сейчас по умолчанию открывается «Заявки».

## Что предлагаю сделать

Доработать админку, чтобы управление товарами было очевидным и удобным:

1. **Видимая навигация в админке**
   - В шапке `/admin` сделать две заметные вкладки: «Заявки» и «Букеты» с подсветкой активной.
   - На `/admin` (заявки) добавить быструю кнопку «+ Новый букет», ведущую сразу в редактор.

2. **Улучшить раздел «Букеты»**
   - Поиск по названию и фильтр по цвету/видимости над сеткой.
   - Inline-переключатель «показывать / скрыть» прямо на карточке (без открытия редактора).
   - Кнопка «Дублировать» — создаёт копию букета как черновик (скрытый), удобно для новых сезонных композиций.
   - Подтверждение удаления через аккуратный модал вместо `confirm()`.

3. **Редактор карточки товара**
   - Автогенерация slug из названия (с возможностью переопределить).
   - Загрузка собственного изображения букета (Lovable Cloud Storage, бакет `product-images`, публичное чтение, запись только админу) — в дополнение к выбору из готовых иллюстраций.
   - Предпросмотр карточки справа от формы: как букет будет выглядеть в каталоге.
   - Базовая валидация (zod): обязательные поля, цена > 0, уникальный slug.

4. **Мелочи UX**
   - Тосты об успехе/ошибке (уже есть) + блокировка повторного сабмита.
   - Сохранение порядка сортировки drag-and-drop опционально — скажите, нужно ли.

## Технические детали

- Фронт: правки в `src/routes/admin/route.tsx` (навигация), `src/routes/admin/products.tsx` (список + действия), вынос `ProductEditor` в отдельный файл, добавление `src/lib/product-schema.ts` (zod).
- Бэкенд: миграция для создания публичного storage-бакета `product-images` и политик (чтение — всем, запись/удаление — только `has_role(admin)`); поле `image_url` уже поддерживает абсолютные URL, отдельных изменений схемы не требуется.
- Дублирование — обычный `insert` с `is_active=false` и суффиксом ` (копия)` в имени и slug.
