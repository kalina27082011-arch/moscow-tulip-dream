План улучшения админ-панели, чтобы сделать её удобной и «полноценной».

## Что есть сейчас
- Две вкладки: «Заявки» и «Букеты».
- Заявки: плоский список, фильтр по статусу, раскрытие карточки, смена статуса.
- Букеты: сетка, поиск, фильтры, редактор, скрыть/показать, дубликат, удаление.
- Нет дашборда, статистики, экспорта, работы с клиентами, загрузки фото, ролей.

## Куда движемся
Админ-панель = 5 разделов с боковым меню (sidebar), общий каркас, единый стиль. Никаких изменений витрины.

```text
┌─ Дашборд ── Заявки ── Букеты ── Клиенты ── Настройки ─┐
│  KPI + график  таблица  сетка   список     доступы    │
└──────────────────────────────────────────────────────────┘
```

## Разделы

### 1. Каркас
- Заменить верхнюю навигацию на shadcn `Sidebar` (сворачиваемый, иконки).
- Хлебные крошки + заголовок страницы в шапке.
- Toaster уже есть, оставить.

### 2. Дашборд (новый `/admin`)
- KPI-карточки: заявок сегодня, выручка сегодня, средний чек, новых за неделю.
- График заявок за 14 дней (recharts, уже в стеке).
- Топ-5 букетов по продажам (агрегируется из `order_items`).
- Последние 5 заявок с быстрым переходом.
- Все данные — из существующих таблиц, без миграций.

### 3. Заявки (`/admin/orders`)
- Перевести список в таблицу (shadcn `Table`) с сортировкой по дате/сумме.
- Поиск по имени/телефону, фильтр по дате (диапазон), статусу.
- Массовое действие: сменить статус нескольким заявкам.
- Детали заявки — в боковой панели (`Sheet`) вместо inline-раскрытия: адрес, время, комментарий, состав, история статусов, кнопка «позвонить» (`tel:`), «скопировать адрес», печать/скачать PDF-накладную (jsPDF на клиенте).
- Экспорт отфильтрованных заявок в CSV.
- Бейдж «новые» в сайдбаре (счётчик заявок в статусе `new`).

### 4. Букеты (`/admin/products`)
- Переключатель «сетка / таблица».
- Инлайн-редактирование цены и порядка сортировки в таблице.
- Drag-and-drop сортировка (dnd-kit) в режиме сетки, сохранение `sort_order` батчем.
- Массовые действия: скрыть/показать/удалить выбранные.
- Загрузка изображения в Supabase Storage (новый bucket `product-images`, публичный read) — сейчас только URL/пресеты.
- В редакторе: счётчики символов, валидация уникальности slug до сохранения, кнопка «открыть на витрине».

### 5. Клиенты (новый `/admin/customers`)
- Агрегируется из `orders` по `phone`: имя, число заказов, общая сумма, последний заказ.
- Клик → список всех заказов клиента.
- Поиск по имени/телефону. Без изменений схемы БД.

### 6. Настройки (новый `/admin/settings`)
- Профиль текущего админа (email, смена пароля).
- Управление админами: список из `user_roles` + приглашение по email (через `supabaseAdmin` в защищённом `createServerFn` с проверкой роли админа).
- Отзыв роли admin у пользователя.

## Технические детали
- **Миграции БД:**
  - Bucket `product-images` (публичный read, insert только для admin).
  - Server function `inviteAdmin`/`revokeAdmin` (createServerFn + requireSupabaseAuth + проверка `has_role`).
  - Опционально: таблица `order_status_history` (order_id, status, changed_by, changed_at) + триггер — для истории статусов. Можно во второй итерации.
- **Пакеты:** `@dnd-kit/core`, `@dnd-kit/sortable`, `jspdf` (уже есть recharts, shadcn — проверить перед `bun add`).
- **Роутинг:** новые файлы `src/routes/admin/orders.tsx`, `customers.tsx`, `settings.tsx`, переезд текущего `admin/index.tsx` в `admin/orders.tsx`, новый дашборд как `admin/index.tsx`.
- Гейт остаётся в `admin/route.tsx` (проверка сессии + роли admin).

## Порядок работ (можно поэтапно)
1. Каркас с сайдбаром + пустые разделы (Дашборд, Клиенты, Настройки).
2. Дашборд с KPI и графиком.
3. Переезд заявок в таблицу + Sheet-детали + CSV.
4. Клиенты (агрегация).
5. Улучшения букетов: инлайн-редакт, DnD, массовые действия, загрузка фото.
6. Настройки: управление админами.

- Нужна ли история статусов заявок (доп. таблица + триггер)?
