
## Что строим

Сайт-магазин тюльпанов с доставкой по Москве за 1 час. Покупатель листает каталог, кладёт букеты в корзину, оставляет заявку без онлайн-оплаты. Владелец заходит в закрытую админку, видит заявки и редактирует каталог.

## Визуальный курс

Лёгкий, весенний, романтичный — но в духе бутика, а не типового цветочного шаблона.

- Палитра: кремовый фон `#FAF6F1`, пыльно-розовый акцент `#E8C5C0`, шалфейно-оливковый `#8A9A82`, графит `#1F1B18` для текста.
- Типографика: заголовки — `Cormorant Garamond` (тёплая serif с характером), тело — `Inter` (нейтральная sans). Крупные h1 с большим интерлиньяжем.
- Композиция: много воздуха, асимметричные секции, фотографии тюльпанов «во весь рост», цены без рамок и ценников.
- Микро-движение: мягкий fade-up при появлении блоков, плавный hover на карточках, никаких баннеров-каруселей и таймеров акций.

После первой сборки покажу превью — там и подкрутим направление.

## Структура сайта (публичная часть)

```text
/                      Главная: hero, превью каталога (6 букетов), блок «доставка за 1 час», блок-история, футер
/catalog               Каталог: сетка букетов с фильтром по цвету/цене
/product/$slug         Карточка товара: фото, описание, состав, цена, кол-во, «в корзину»
/cart                  Корзина: список позиций, итог, форма заявки (имя, телефон, адрес, время, комментарий)
/delivery              Условия доставки
/privacy               Политика конфиденциальности
/offer                 Публичная оферта
/admin/login           Вход владельца
/admin                 Дашборд: список заявок
/admin/products        Управление товарами (список + создание + редактирование)
```

Хедер (sticky): логотип-вордмарк, ссылки Каталог / Доставка / Контакты, иконка корзины со счётчиком.
Футер: контакты, часы работы, ссылки на Доставку / Политику / Оферту, соцсети.

## Ключевые экраны

- **Главная** — hero «Тюльпаны с доставкой за час по Москве», крупное фото, CTA «Смотреть каталог». Ниже — 6 букетов превью, секция «Как мы доставляем за час» (3 шага без иконок-стоков, набрано типографикой), секция-история ателье, форма-CTA.
- **Каталог** — отзывчивая сетка 2/3 колонки, лёгкий фильтр (цвет, цена, размер букета), карточка показывает фото-цену-название, при hover проявляется кнопка «в корзину».
- **Карточка товара** — крупное фото слева, справа название/цена/состав/описание/количество/CTA. Внизу «может понравиться» (3 букета).
- **Корзина** — позиции с превью, изменение количества, удаление, итог. Форма заявки прямо на этой же странице: имя, телефон, адрес, желаемое время, комментарий. Кнопка «Оформить заявку» — заявка падает в админку, покупатель видит экран «спасибо, мы перезвоним в течение нескольких минут».

## Админка владельца

- Аутентификация через Lovable Cloud (email + пароль). Роль `admin` хранится в отдельной таблице `user_roles`, проверяется security-definer функцией `has_role`.
- Первого админа назначаю руками через миграцию после того, как владелец зарегистрируется (либо seed-юзер, обсудим email отдельно).
- `/admin` — таблица заявок (новые/в работе/выполнено/отменено), переход в карточку заявки, смена статуса, заметки.
- `/admin/products` — CRUD товаров: название, slug, цена, описание, состав, активность, фото (загрузка в Lovable Storage), сортировка.
- Все админские маршруты под `_authenticated` layout с редиректом неавторизованных на `/admin/login`, плюс проверка роли `admin` внутри.

## Данные (Lovable Cloud)

```text
products       id, slug, name, price, description, composition, image_url, is_active, sort_order, created_at
orders         id, customer_name, phone, address, delivery_time, comment, total, status, created_at
order_items    id, order_id, product_id, name_snapshot, price_snapshot, qty
user_roles     id, user_id (-> auth.users), role ('admin')
```

RLS:
- `products`: публичный SELECT активных товаров для `anon`; полный доступ — только админам.
- `orders` / `order_items`: INSERT разрешён всем (создание заявки), SELECT/UPDATE — только админам.
- `user_roles`: SELECT только аутентифицированным через `has_role`.

Все GRANT'ы прописываю в той же миграции.

## Технические детали

- TanStack Start (текущий шаблон), TanStack Router file-based routing, TanStack Query для данных.
- Шрифты — через `@fontsource/cormorant-garamond` и `@fontsource/inter`, регистрация в `src/styles.css` через `@theme`.
- Дизайн-токены — в `src/styles.css` через `@theme inline` + CSS-переменные. Никаких хардкод цветов в компонентах.
- Корзина — в `localStorage` + zustand-store (без таблицы корзин, чтобы не плодить сущности).
- Создание заявки — `createServerFn` с Zod-валидацией, пишет в `orders` + `order_items` через service-role клиент (анон INSERT тоже допустим, но через server fn чище).
- Управление товарами — `createServerFn` под `requireSupabaseAuth` + проверка роли `admin`.
- Картинки тюльпанов — генерирую через `imagegen` в `src/assets/` для seed-каталога (6-8 букетов) с романтичными названиями типа «Майя», «Серенада», «Утро в Замоскворечье».
- SEO: каждая страница со своим `head()` (title, description, og-теги). Главная и карточка товара — с `og:image`.

## Порядок сборки

1. Дизайн-система: токены, шрифты, базовые компоненты (Button, Input, Container).
2. Хедер, футер, layout.
3. Включаю Lovable Cloud, накатываю схему + RLS + GRANT'ы, генерирую seed-фотки и заливаю тестовые товары.
4. Главная, каталог, карточка товара.
5. Корзина + форма заявки + server-fn создания заказа.
6. Страницы Доставка / Политика / Оферта.
7. Аутентификация + роли + админка (заявки + товары).
8. Прогон сборки, фикс багов, проверка через Playwright (создание заявки, вход в админку).
